빛으로 연산하는 칩이 나왔어요. 전자 대신 광자를 쓰는 시대가 열립니다

• NIST 연구팀이 리튬 나이오베이트와 탄탈라를 실리콘 웨이퍼 위에 3차원으로 쌓아 빛으로 정보를 처리하는 포토닉스 칩을 만들었어요. 손톱만 한 칩 하나에 1만 개의 광학 회로가 들어가고, 웨이퍼 한 장에 50개를 동시에 뽑아낼 수 있죠.
• 탄탈라 소재 덕분에 레이저 한 가지 색을 가시광선 전체와 적외선까지 변환할 수 있어요. AI 추론, 양자 컴퓨터, 원자시계처럼 엄청난 연산 정밀도가 필요한 분야에서 게임 체인저가 될 수 있는 기술이에요.
• 스타트업 Octave Photonics와 협력해 양산 가능성을 타진 중이에요. 아직 대량 생산까지는 넘어야 할 허들이 남아 있지만, 전자 기반 반도체의 물리적 한계를 광학으로 우회하려는 시도라는 점에서 주목할 만해요.

WAVE 02 221 pts · 69 comments

1960년대 B-52 폭격기는 별을 보며 길을 찾았어요. 그 안에 숨겨진 아날로그 컴퓨터 이야기

• 냉전 시대 B-52에는 Astro Compass라는 자동 천체 항법 시스템이 달려 있었어요. 별의 방위각과 고도를 계산하는 전기기계식 앵글 컴퓨터가 천구를 물리적으로 모델링해서, 항법사가 손으로 삼각함수를 풀어야 했던 일을 자동화했죠.
• 시스템은 자이로스코프, 광전자 증배관, 싱크로 회전 센서 등 19개 부품으로 구성됐어요. 항법사가 항성시각, 적위, 시간 같은 값을 로터리 노브로 입력하면 기계가 별 세 개를 동시에 추적하며 0.1도 정밀도로 방향을 잡았어요.
• GPS도, 인터넷도 없던 시절에 재밍이 불가능하고 외부 인프라도 필요 없는 항법 시스템을 만들어낸 거예요. 소프트웨어 없이 기어와 모터만으로 구현한 연산 로직이라는 점에서 지금 봐도 놀랍죠.

💡 오늘의 인사이트

WAVE 03 163 pts · 110 comments

Figma의 시대가 끝나가고 있어요. 코드가 다시 디자인의 원본이 되고 있거든요

• Figma는 독자적인 파일 포맷을 고집했는데, 그게 오히려 발목을 잡았어요. LLM 학습 데이터에서 배제되면서 AI 에이전트 시대에 자연스럽게 연동되기 어려운 구조가 됐죠. 반면 Claude Design처럼 코드를 직접 다루는 도구들은 AI와 궁합이 잘 맞아요.
• Figma의 중첩 컴포넌트, 변수, 앨리어스, 모드, 오버라이드 시스템은 디자인을 체계화하려고 만든 건데, 지금은 오히려 디버깅 악몽이 됐어요. AI가 코드를 직접 생성하고 수정할 수 있게 되면서, 디자인 파일이 아니라 코드가 다시 진실의 원본이 되는 흐름이 만들어지고 있어요.
• 디자인 툴링은 두 갈래로 나뉠 것 같아요. Figma Make처럼 디자인 파일을 정본으로 유지하는 방향과, Claude Design처럼 코드를 기본 매체로 삼는 방향이에요. 어느 쪽이 이길지는 아직 모르지만, 흐름은 분명히 후자 쪽으로 기울고 있어요.

💡 오늘의 인사이트

프론트엔드 개발자라면 이 변화가 남의 일이 아니에요. 디자이너와 협업하는 방식, 컴포넌트를 관리하는 방식, 디자인 시스템을 운영하는 방식 모두 영향을 받을 수 있어요. 지금 당장 Figma를 버릴 필요는 없지만, 코드 기반 디자인 워크플로우가 어떻게 돌아가는지 한번쯤 직접 써보는 게 좋아요. 변화는 항상 써본 사람이 먼저 알아채거든요.

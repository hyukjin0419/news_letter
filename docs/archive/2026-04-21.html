오픈소스 AI 모델, 벤더마다 성능이 다르다고요? 이제 직접 검증할 수 있어요

• Kimi가 K2.6 모델과 함께 오픈소스 검증 도구 Kimi Vendor Verifier (KVV)를 공개했어요. 같은 모델인데 벤더마다 벤치마크 점수가 다르게 나오는 이상한 경험, 한 번쯤 해보셨죠? 이게 모델 문제가 아니라 배포 환경 문제일 수 있다는 걸 이 프로젝트가 증명해줘요.
• 원인 중 하나는 Temperature와 TopP 같은 디코딩 파라미터 오설정이었어요. Kimi는 이제 Thinking 모드에서 Temperature=1.0, TopP=0.95를 API 레벨에서 강제 적용해요. vLLM, SGLang, KTransformers 커뮤니티와 함께 업스트림 수정도 병행하고 있고요.
• 검증 파이프라인은 NVIDIA H20 GPU 8장짜리 서버 두 대에서 15시간 순차 실행으로 돌아가요. 공개 리더보드도 함께 운영해서 벤더별 성능을 지속적으로 비교할 수 있게 했어요. 모델 능력 문제인지 구현 실패인지 이제 구분할 수 있게 됐죠.

💡 오늘의 인사이트

오픈소스 모델을 프로덕션에 붙일 때 가장 무서운 게 뭔지 아세요? 내가 테스트한 성능이랑 실제 서비스 성능이 다를 때예요. KVV 같은 도구가 생긴다는 건, 이제 모델 선택의 기준이 벤치마크 숫자에서 검증된 배포 환경으로 옮겨간다는 뜻이에요. 팀에서 오픈소스 LLM을 도입하고 있다면, 어떤 벤더의 추론 서버를 쓰는지도 꼭 확인해보세요.

깃허브 별 6백만 개가 가짜였다고요, 그럼 우리는 뭘 믿어야 하죠

• CMU 연구팀이 2019년부터 2024년 사이 깃허브 메타데이터 20TB를 분석했어요. 결과는 충격적이에요. 18,617개 저장소에 달린 별 약 600만 개가 가짜였고, 이걸 단 30만 1천 개 계정이 만들어냈어요. 별 하나 가격은 $0.03에서 $0.85 사이, 공개 사이트에서 대놓고 팔리고 있었죠.
• AI와 LLM 관련 저장소가 비악성 카테고리 중 가짜 별을 가장 많이 받은 분야예요. 무려 177,000개나 됩니다. 2024년 7월 기준으로 별 50개 이상인 저장소의 16.66%가 가짜 별 캠페인에 연루돼 있었어요. 2022년 이전엔 거의 없던 일이에요.
• VC들이 투자 후보를 고를 때 깃허브 별 수를 명시적 지표로 쓴다는 게 확인됐어요. Redpoint 기준 시드 단계 중간값이 2,850개 별이에요. 별 구매 비용은 최소 $0.06인데, 이게 시드 투자 수백만 달러로 이어질 수 있다는 거죠. StarScout 같은 탐지 도구와 깃허브의 자체 삭제 조치로 플래그된 저장소의 90.42%가 사라졌지만, 생태계 신뢰는 이미 흔들렸어요.

Git 이후의 버전 관리가 온다: GitHub 공동창업자가 170억을 받은 이유

 
• GitHub 공동창업자 Scott Chacon이 만든 GitButler가 a16z 주도로 170억 원 규모의 시리즈 A 투자를 받았어요. 'Git을 현대 개발 방식에 맞게 다시 설계하겠다'는 한 문장이 투자자들을 움직인 거죠.
• GitButler CLI 기술 미리보기가 공개됐어요. GitHub Flow 기반의 단명 브랜치 워크플로우에 최적화되어 있고, 기존 Git 프로젝트와 그대로 호환돼요. 브랜치 쌓기, 멀티태스킹, 쉬운 되돌리기 기능을 품고 있어요.
• 핵심 문제의식은 이거예요. 코드 변경사항을 정리하고, 팀원과 맥락을 공유하고, AI 에이전트와 함께 작업하는 흐름을 Git이 제대로 받쳐주지 못한다는 것. GitButler는 사람과 AI가 함께 일하는 팀 협업 도구를 목표로 해요.

💡 오늘의 인사이트

 

솔직히 말하면, Git은 위대하지만 불편하죠. 브랜치 전환하다 작업 날려먹고, PR 쌓이면 맥락 잃어버리고, AI 에이전트가 커밋 올리면 뭐가 뭔지 모르게 되는 그 느낌 다들 알 거예요. GitButler가 그 고통을 진짜로 해결해줄지는 아직 지켜봐야 하지만, GitHub을 만든 사람이 직접 나섰다는 건 무시하기 어려운 신호예요. CLI 기술 미리보기가 공개됐으니, 사이드 프로젝트 하나에 슬쩍 얹어보는 건 어떨까요. 어차피 Git 쓰던 프로젝트에 그대로 붙여볼 수 있으니 부담도 없어요.

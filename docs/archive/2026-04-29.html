Ghostty가 GitHub을 떠난다고요? 18년 만에 짐을 싼 이유

• 터미널 에뮬레이터 Ghostty의 개발자 Mitchell Hashimoto가 GitHub 이전을 선언했어요. 이유는 단순해요. 지난 한 달 동안 거의 매일 장애가 있었고, GitHub Actions 다운으로 PR 리뷰조차 못 하는 날이 반복됐거든요.
• 프로젝트는 아직 검토 중인 대안 플랫폼으로 점진적으로 옮기고, GitHub에는 읽기 전용 미러만 남겨둘 계획이에요. 개인 프로젝트는 당분간 GitHub에 유지하고요.
• 2008년부터 18년을 함께한 플랫폼을 떠나는 결정이라, 단순한 불만이 아니라 몇 달에 걸친 고민 끝에 내린 선택이에요. 특정 장애 하나가 방아쇠를 당긴 게 아니라, 쌓인 신뢰 손상이 원인이었죠.

💡 오늘의 인사이트

CI/CD 파이프라인이 플랫폼 장애 하나로 멈춰버린 경험, 다들 한 번쯤 있죠. 이번 사건은 단순히 '깃허브 또 터졌네'로 넘길 게 아니에요. 우리 팀의 개발 워크플로우가 단일 플랫폼에 얼마나 깊이 묶여 있는지 점검해볼 좋은 계기예요. 미러 저장소 하나 만들어두는 것, 생각보다 어렵지 않아요.

GitHub 이전 시대를 기억하는 사람이 쓴 글, 읽어볼 만해요

• GitHub 등장 전 오픈소스 세상은 SourceForge, Subversion, 자체 호스팅 서버가 주류였어요. 진입 장벽이 높았고, 프로젝트는 쌓인 평판 위에서 움직였죠. GitHub은 이 모든 마찰을 없애버렸고, 그 결과 npm 생태계처럼 마이크로 의존성이 폭발적으로 늘어났어요.
• 문제는 GitHub이 이제 단순한 코드 저장소가 아니라, 오픈소스 커뮤니티의 사회적 인프라가 됐다는 거예요. 이슈 히스토리, 기여자 맥락, 프로젝트의 '이야기'가 모두 한 회사 서버 안에 있어요.
• 저자는 Microsoft 소유 이후 GitHub의 장기적 건강을 걱정하며, 앞으로의 오픈소스 인프라는 프로젝트 이식성과 사회적 맥락 보존을 최우선으로 설계해야 한다고 말해요.

💡 오늘의 인사이트

의존성 하나가 사라지면 npm 생태계가 흔들렸던 사건들, 기억하시죠. 플랫폼도 마찬가지예요. 우리가 GitHub에 올리는 건 코드만이 아니에요. 팀의 의사결정 기록, 커뮤니티와의 대화, 프로젝트의 역사가 함께 쌓이고 있어요. 지금 당장 이전할 필요는 없지만, 그 데이터를 주기적으로 백업하고 있는지는 한번 확인해보세요.

OpenAI가 AWS에 올라탔어요. Azure 독점 시대가 끝났습니다

• Microsoft와 OpenAI가 계약을 수정했어요. 이제 OpenAI 모델은 Azure 외 다른 클라우드에서도 서비스할 수 있어요. 그 첫 번째 행선지가 AWS Bedrock이에요. Microsoft는 여전히 1순위 파트너이고 2032년까지 IP 라이선스를 유지하지만, 독점권은 사라졌어요.
• OpenAI는 Bedrock Managed Agents를 AWS에서 출시해요. 기업들이 이미 쓰고 있는 AWS 인프라 위에서 AI 에이전트 워크플로우를 바로 구성할 수 있게 되는 거죠. AWS Trainium 칩 연동도 논의 중이에요.
• 독점 구조가 OpenAI 성장의 발목을 잡고 있었다는 게 이번 변화의 핵심 배경이에요. 기업 고객들이 기존 클라우드 벤더를 바꾸지 않으려는 성향이 강하다 보니, OpenAI 입장에서는 어디서든 만날 수 있어야 했던 거예요.

💡 오늘의 인사이트

AWS를 주로 쓰는 팀이라면 반가운 소식이에요. 이제 Bedrock 안에서 OpenAI 모델을 직접 쓸 수 있게 되면, 클라우드 벤더를 바꾸지 않아도 AI 기능을 붙일 수 있는 선택지가 늘어나는 거니까요. 다만 여러 클라우드에 걸쳐 AI 모델이 분산되기 시작하면, 비용 관리와 모델 버전 추적이 새로운 두통거리가 될 수 있어요. 미리 거버넌스 기준을 잡아두는 게 좋아요.

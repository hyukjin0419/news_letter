비트워든 CLI를 쓰고 있다면 지금 당장 확인하세요

• @bitwarden/cli 버전 2026.4.0에 악성 파일 bw1.js가 심어졌어요. GitHub Actions CI/CD 파이프라인이 뚫리면서 생긴 일인데, GitHub 토큰, AWS, Azure, GCP 키, SSH 키까지 몽땅 긁어가는 구조예요. npm 패키지만 영향을 받고, 크롬 익스텐션이나 다른 배포판은 괜찮아요.
• 이 악성코드는 훔친 npm 토큰으로 다른 패키지에도 퍼지고, GitHub Actions 워크플로우에 자신을 주입해서 살아남아요. 더 섬뜩한 건 ~/.bashrc와 ~/.zshrc에 영속성까지 심는다는 거예요. C2 서버 주소가 audit.checkmarx[.]cx인데, 시스템 로케일이 ru로 시작하면 조용히 종료되는 킬 스위치도 들어있어요.
• 지금 당장 해야 할 일은 세 가지예요. 패치된 버전으로 업데이트하고, GitHub Actions 로그를 꼼꼼히 살펴보고, 관련 시크릿을 전부 교체하는 거예요. 1,000만 명 이상이 쓰는 도구가 공급망 공격에 당했다는 건, 우리 팀 파이프라인도 예외가 아니라는 뜻이에요.

클로드가 갑자기 멍청해진 것 같았던 그 느낌, 실제였어요

• 3월에서 4월 사이에 Claude Code가 뭔가 이상하다고 느꼈다면 착각이 아니에요. Anthropic이 공식으로 인정한 세 가지 별개의 문제가 동시에 터졌거든요. 가장 큰 건 Opus 4.6의 기본 노력 수준이 high에서 medium으로 몰래 바뀐 거예요. 쉽게 말하면 모델이 갑자기 대충 생각하기 시작한 거죠.
• 두 번째 문제는 프롬프트 캐싱 효율화 작업이 원인이었어요. 3월 26일 변경으로 대화 히스토리에서 추론 과정이 빠지면서, 뒤로 갈수록 맥락을 못 잡는 현상이 생겼어요. 내부 평가 지표에서는 안 잡혔는데 실제 사용자들은 바로 느꼈다는 게 포인트예요.
• Anthropic은 4월 20일 v2.1.116으로 전부 수정했고, 4월 23일에는 모든 구독자 사용량을 리셋했어요. 앞으로는 단계적 배포, 더 넓은 평가 세트, 프롬프트 변경 리뷰 프로세스를 도입한다고 해요.

AI 코드로 팀원 몰래 프로젝트 절반을 갈아엎으면 어떻게 될까요

• 메시 네트워킹 오픈소스 프로젝트 MeshCore가 내부 분쟁으로 쪼개졌어요. 팀원 Andy Kirby가 Claude로 독립 디바이스, 모바일 앱, 웹 플래셔, 웹 설정 도구를 통째로 재작성했는데, 이걸 팀에 알리지 않았어요. 거기다 3월 29일에 MeshCore 상표까지 팀 동의 없이 혼자 등록해버렸어요.
• 핵심 팀은 Andy가 실제로 기여한 GitHub 저장소가 없다는 점을 들어 공식 프로젝트는 자신들이라고 선언했어요. 그리고 meshcore.io를 새 공식 사이트로 열었는데, Andy가 또 Claude로 그 디자인을 그대로 복제해버렸어요. 현재 전 세계 38,000개 이상의 노드, 100,000명 이상의 앱 사용자가 어느 쪽이 진짜인지 혼란스러운 상황이에요.
• 이 사건은 기술적 문제가 아니라 거버넌스 문제예요. AI로 빠르게 만든 코드가 프로덕션에 들어갔고, 기여 범위와 소유권에 대한 합의가 없었던 게 결국 법적 분쟁으로 번졌어요.

💡 오늘의 인사이트

AI 코딩 도구가 일상이 된 지금, 팀 안에서 이런 질문을 미리 해두면 좋아요. AI로 작성한 코드의 기여자는 누구인지, 팀 동의 없이 핵심 컴포넌트를 재작성해도 되는지, 프로젝트 관련 상표나 도메인은 누가 어떻게 관리하는지요. 사소해 보이지만, MeshCore처럼 잘 되던 프로젝트가 이 합의 없이 무너지는 데는 몇 달이면 충분하더라고요.

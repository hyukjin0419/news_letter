VS Code 말고 Zed, 이제 진짜 써볼 때가 됐어요

• Zed 1.0이 정식 출시됐어요. Electron 기반이 아니라 Rust로 처음부터 다시 만든 에디터예요. GPU로 UI를 직접 렌더링하는 GPUI 프레임워크 덕분에, 코드가 10만 줄이 넘어도 버벅임 없이 부드럽게 돌아가요.
• AI가 나중에 붙은 게 아니라 에디터 설계 자체에 녹아 있어요. Claude Agent, Codex, Cursor 같은 에이전트를 동시에 여러 개 띄워서 병렬로 작업을 맡길 수 있어요. 탭 하나에서 리팩토링하면서 다른 에이전트가 테스트를 짜는 식이죠.
• 실시간 협업을 위한 DeltaDB도 개발 중이에요. CRDT 기반이라 사람과 AI 에이전트가 같은 파일을 동시에 수정해도 충돌 없이 합쳐져요. 구글 독스처럼 코드를 같이 쓰는 날이 멀지 않았어요.

💡 오늘의 인사이트

Electron 에디터에 익숙해진 탓에 우리가 얼마나 많은 메모리와 시간을 낭비하고 있었는지, Zed를 한 번만 써보면 바로 느껴져요. 물론 완전히 독자적인 스택을 선택한 만큼 생태계가 아직 얇고, GPUI나 DeltaDB 같은 독자 기술에 대한 의존도가 높아지는 건 리스크예요. 당장 갈아타기보다는, 사이드 프로젝트에서 하루 써보는 걸 추천해요. 속도 차이를 몸으로 느끼고 나면 선택이 훨씬 쉬워질 거예요.

WAVE 02 449 pts · 212 comments

732바이트 파이썬 스크립트 하나로 리눅스 루트를 따낸다고요

• CVE-2026-31431, 일명 Copy Fail이에요. 2017년 이후 빌드된 거의 모든 리눅스 배포판에 영향을 줘요. Ubuntu 24.04 LTS, Amazon Linux 2023, RHEL 등 여러분 팀 서버에 깔린 그 운영체제 맞아요.
• 무서운 건 조건이 너무 단순하다는 거예요. 네트워크 접근도, 커널 디버그 기능도 필요 없어요. 로컬에서 일반 사용자 계정 하나만 있으면 돼요. 커널 암호화 API인 AF_ALG와 splice() 사이의 로직 결함을 이용해서 페이지 캐시에 4바이트를 덮어쓰는 방식이에요. 레이스 컨디션도 없어서 거의 확실하게 터져요.
• 조치 방법은 두 가지예요. 커널을 메인라인 커밋 a664bf3d603d 이후 버전으로 올리거나, /etc/modprobe.d/에 설정 파일을 추가해서 algif_aead 모듈을 비활성화하면 돼요. PoC 스크립트가 이미 공개됐으니 빠를수록 좋아요.

💡 오늘의 인사이트

CI/CD 러너, 공용 개발 서버, 멀티테넌트 환경이라면 지금 당장 팀 채널에 공유해야 해요. 혼자만 쓰는 단독 서버라면 상대적으로 위험이 낮지만, 로컬 코드 실행이 곧 루트가 된다는 사실은 변하지 않아요. 인프라 담당 동료에게 커피 한 잔 건네면서 오늘 안에 패치 일정 잡아달라고 부탁해 보세요. 나중에 인시던트 리포트 쓰는 것보다 훨씬 나아요.

WAVE 03 922 pts · 370 comments

커밋 메시지에 HERMES.md 썼다가 요금 폭탄 맞은 사람들 이야기

• Claude Code를 쓰는 팀에서 커밋 메시지나 코드 주석에 특정 문자열이 포함되면 요청이 추가 과금 경로로 라우팅된다는 게 알려졌어요. 의도치 않게 비싼 모델 호출이 발생하는 거예요. 자동화된 워크플로에서 조용히 돈이 빠져나가는 구조라 더 까다로워요.
• Claude Code는 Node.js 18+ 기반의 터미널 에이전트예요. 자연어 명령으로 코드 설명, 리팩토링, 깃 워크플로를 처리해 줘요. GitHub 연동도 되고, 커스텀 명령어로 기능을 확장할 수 있어서 팀 도구로 붙이기 좋은 구조예요.
• 다만 사용 데이터 수집 범위가 꽤 넓어요. 코드 수락 및 거절 기록, 대화 내용, 사용자 피드백이 모두 수집돼요. 사내 코드베이스에 붙이기 전에 서비스 약관과 개인정보 처리 방침을 팀 전체가 한 번은 읽어볼 필요가 있어요.

💡 오늘의 인사이트

AI 코딩 도구가 팀 워크플로에 깊이 들어올수록, 비용과 데이터 흐름을 눈에 보이게 관리하는 게 중요해요. 개인 토큰으로 쓰다가 팀 전체로 확장하는 순간, 요금 구조와 데이터 정책이 완전히 다른 문제가 돼요. 도입 전에 샌드박스에서 어떤 데이터가 어디로 나가는지 한 번 확인해 두면, 나중에 보안 팀에 설명하는 수고를 덜 수 있어요.

Git 이후의 버전 관리가 온다, GitHub 공동창업자의 새 도전

 
• GitHub 공동창업자 Scott Chacon이 만든 GitButler가 a16z 주도로 170억 원 규모의 시리즈 A 투자를 받았어요. 단순히 돈을 많이 받았다는 얘기가 아니라, '버전 관리 자체를 다시 설계하겠다'는 꽤 큰 선언이에요.
• 기술 프리뷰로 공개된 GitButler CLI는 GitHub Flow 기반 워크플로우에 최적화되어 있어요. branch stacking과 멀티태스킹을 지원하고, 기존 Git 프로젝트에 바로 붙여 쓸 수 있어서 도입 장벽이 낮아요. 브랜치 쌓아두다가 머지 지옥 경험해본 분들이라면 눈이 번쩍 뜨일 기능이에요.
• AI 에이전트와의 협업을 염두에 두고 설계됐다는 점도 주목할 만해요. 코드를 생성하는 것보다 변경사항을 정리하고 리뷰하고 합치는 과정, 즉 우리가 매일 시간을 갉아먹히는 그 부분을 줄이는 데 집중하고 있어요.

💡 오늘의 인사이트

 

솔직히 Git은 여전히 강력하지만, 매일 쓰면서 '이게 최선인가' 싶은 순간이 있잖아요. 브랜치 이름 짓다가 멈추거나, 리베이스 꼬여서 30분 날리거나. GitButler가 그 불편함을 정말 해결해주는지는 써봐야 알겠지만, 기술 프리뷰 단계부터 기존 Git 프로젝트에 바로 붙여 쓸 수 있다는 건 일단 시도해볼 이유가 충분해요. 큰 결심 없이 지금 쓰는 프로젝트에 살짝 얹어보는 것, 오늘 점심 시간에 해볼 만한 일이에요.

레고와 듀플로를 연결하는 어댑터를 직접 출력할 수 있다면?

• F.A.T. Lab이 레고, 듀플로, K'Nex 등 10가지 장난감 시스템을 연결하는 어댑터 80여 종을 무료 .STL 파일로 공개했어요. Thingiverse에서 내려받아 3D 프린터로 바로 출력할 수 있죠.
• 이 프로젝트의 핵심은 '역설계를 시민 활동으로 본다'는 관점이에요. 상업 제품의 규격 장벽을 오픈소스로 허무는 방식인데, 개발자라면 API 호환성 문제를 풀 때 이 접근법이 꽤 익숙하게 느껴질 거예요.
• 장난감 수명을 연령대에 관계없이 늘린다는 실용적인 가치도 있지만, 독점 커넥터 규격을 역설계한 만큼 지식재산권 이슈는 조용히 따라붙어요.

💡 오늘의 인사이트

레거시 시스템 두 개를 연결하는 어댑터 레이어를 짜본 적 있다면, 이 프로젝트가 그냥 장난감 얘기로 안 들릴 거예요. 호환되지 않는 두 세계 사이에 얇은 다리 하나를 놓는 일, 그게 결국 우리가 매일 하는 일이기도 하니까요.

512 x 342 픽셀로 호쿠사이의 파도를 다시 그리는 사람

• 한 아티스트가 호쿠사이의 '후지산 36경' 전작을 1비트 픽셀 아트로 재현하는 프로젝트를 5년째 이어가고 있어요. 도구는 Macintosh Quadra 700과 Aldus SuperPaint 3.0, 해상도는 딱 512 x 342픽셀로 고정이에요.
• 현대 툴을 쓰지 않고 당시 하드웨어와 소프트웨어만 고집하는 이유는 '제약이 곧 미학'이기 때문이에요. 수잔 케어가 초기 맥 아이콘을 디자인하던 그 감각을 그대로 따라가는 거죠.
• 작업물은 Creative Commons Attribution-NonCommercial-NoDerivatives 4.0으로 공개되어 있고, 640 x 480 맥 화면용 바탕화면 버전도 있어요. 다만 프로젝트는 현재 멈춰 있고 재개 시점은 불확실해요.

💡 오늘의 인사이트

최신 GPU도, 생성 AI도 없이 수십 년 된 컴퓨터 한 대로 이 작업을 하는 사람을 보면 잠깐 멈추게 돼요. 도구의 한계를 탓하기 전에, 그 안에서 뭘 만들 수 있는지 먼저 생각해보는 태도, 개발자한테도 꽤 필요한 자세예요.

WAVE 03 123 pts · 88 comments

AI로 '어차피 안 끝낼 프로젝트'를 하룻저녁에 완성한 방법

• 저자는 유튜브 뮤직을 OpenSubsonic API 규격으로 노출하는 심(shim) 서비스를 Claude Code (Opus 4.6)로 하룻저녁에 완성했어요. 핵심은 AI에게 맡기기 전에 CLAUDE.md에 의존성, 코딩 컨벤션, OpenAPI 스펙을 명확히 정리해둔 거예요.
• 저자는 개인 프로젝트를 두 종류로 나눠요. 배우려고 시작한 프로젝트와, 그냥 결과물이 필요한 소원 성취형 프로젝트. AI 도움은 후자에만 쓰는 게 맞다고 해요. 전자까지 AI에 맡기면 실력이 녹아내리니까요.
• 기술 스택은 FastAPI, Pydantic V2, ytmusicapi, yt-dlp, uv로 구성됐고, Navidrome이나 Feishin 같은 음악 클라이언트와 바로 연결돼요.

💡 오늘의 인사이트

폴더 어딘가에 반쯤 만들다 멈춘 프로젝트 하나쯤은 다들 있잖아요. 그게 배움을 위한 프로젝트라면 끝까지 직접 씨름해야 해요. 하지만 그냥 '이게 있으면 좋겠다'는 마음으로 시작한 거라면, AI한테 넘기는 게 오히려 솔직한 선택일 수 있어요. 중요한 건 그 둘을 구분하는 눈이에요.

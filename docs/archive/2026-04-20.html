반도체 공장이 멈추는 건 코드 문제가 아닐 수도 있어요

• 우리가 매일 쓰는 DRAM과 NAND 플래시 칩을 만들려면 브롬화수소 가스가 꼭 필요해요. 그런데 한국이 수입하는 브롬의 97.5%가 이스라엘 단 한 곳에서 와요. 이란 미사일이 이미 그 공장 35km 안까지 떨어졌고요.
• 브롬화수소는 폴리실리콘 식각 공정에서 염소 계열 대비 선택비가 100:1로 압도적이라 대체가 안 돼요. 다른 나라에서 반도체 등급으로 만들 수 있는 설비 자체가 지금 지구상에 없어요.
• 새 전환 설비를 짓는 데는 허가부터 자격 인증까지 수년이 걸려요. 공급이 끊기면 소비자 기기부터 군사 시스템까지 몇 주 안에 타격이 와요. 소프트웨어 스택 아래 하드웨어, 그 아래 소재, 그 아래 지정학이 있다는 걸 오늘 다시 느끼게 돼요.

💡 오늘의 인사이트

우리는 보통 의존성이라고 하면 package.json을 떠올리죠. 그런데 진짜 무서운 의존성은 코드 바깥에 있을 때가 많아요. 공급망 리스크는 인프라 엔지니어만의 고민이 아니에요. 내가 만드는 서비스가 어떤 물리적 토대 위에 서 있는지 가끔은 한 단계 더 내려다보는 시각이 필요해요.

WAVE 02 426 pts · 272 comments

Vercel 보안 사고, 내 토큰은 안전한가요

• Vercel이 내부 시스템 일부에 무단 접근이 있었다고 공식 인정했어요. 침입 경로는 직접 해킹이 아니라 직원들이 쓰던 서드파티 AI 도구의 Google Workspace OAuth 앱이 탈취된 거예요. 내 계정이 아니라 내가 연결해둔 앱이 문제가 된 거죠.
• 공격자는 NPM 토큰, GitHub 토큰, API 키, 직원 580명 정보, 내부 배포 접근권을 가져갔다고 주장해요. 랜섬으로 200만 달러를 요구했다는 이야기도 나왔어요. Vercel은 핵심 서비스는 영향 없다고 하지만, 고객사 환경변수는 직접 점검을 권고하고 있어요.
• 지금 당장 할 수 있는 일이 있어요. Vercel 대시보드에서 환경변수를 확인하고, 연결된 OAuth 앱 목록을 검토하고, 오래된 토큰은 교체하는 거예요. 특히 GitHub과 NPM 토큰은 오늘 바로 돌려보는 게 좋아요.

💡 오늘의 인사이트

이번 사고에서 제일 무서운 건 Vercel이 뚫린 게 아니에요. Vercel 직원이 쓰던 AI 도구가 뚫린 거예요. 우리 팀도 마찬가지예요. 편하다고 연결해둔 서드파티 앱 하나가 전체 배포 파이프라인의 구멍이 될 수 있어요. 팀 내 OAuth 연동 현황을 한 번쯤 같이 점검해보는 시간을 만들어보는 건 어떨까요.

스위스가 Microsoft에 1조 넘게 쓰고 나서 내린 결론

• 스위스 연방 정부가 행정 워크스테이션 54,000대에 Microsoft 365를 깔았어요. 그러면서 동시에 장기적으로 Microsoft 의존도를 줄이겠다고 선언했어요. 지난 10년간 마이크로소프트 라이선스에만 11억 스위스 프랑, 우리 돈으로 1조 원이 넘게 썼거든요.
• 핵심 이유는 미국의 Cloud Act예요. 2018년에 만들어진 이 법은 서버가 어느 나라에 있든 미국 기업이 보유한 데이터를 미국 정부가 요청할 수 있게 해요. 스위스 땅에 데이터를 저장해도 법적으로 미국 당국 손에 닿을 수 있다는 뜻이에요.
• 독일 슐레스비히홀슈타인 주는 이미 행정 전체를 오픈소스로 전환했어요. 스위스 타당성 조사도 전환이 가능하다고 결론 냈어요. 당장 바꾸는 게 아니라 단계적으로, 그리고 장기적으로 가겠다는 방향이에요.

💡 오늘의 인사이트

우리 팀 인프라도 한번 들여다볼 만해요. 지금 쓰는 SaaS 도구들이 어느 나라 법의 적용을 받는지, 고객 데이터가 어디에 저장되는지 정확히 알고 있나요. 규정 준수나 보안 감사 때 갑자기 이 질문이 나오면 당황스럽거든요. 스위스처럼 당장 바꾸지 않더라도, 지금 쓰는 도구의 법적 위치 정도는 파악해두는 게 좋아요.

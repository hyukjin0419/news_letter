예측 시장에서 항상 '아니오'에 베팅하는 봇을 만든 사람이 있어요

• '세상에 별일은 잘 안 일어난다'는 철학을 코드로 구현한 Python 봇이에요. Polymarket의 스포츠 외 예측 시장에서 설정한 가격 이하의 'No' 포지션을 자동으로 매수해요. 대시보드, 포지션 추적, 상태 복구까지 갖춘 꽤 진지한 프로젝트죠.
• 실제 거래가 실행되려면 BOT_MODE=live, LIVE_TRADING_ENABLED=true, DRY_RUN=false 세 가지 환경 변수가 동시에 설정돼야 해요. 하나라도 빠지면 자동으로 PaperExchangeClient로 전환돼서 모의 거래만 해요. 이 안전장치 설계가 꽤 인상적이에요.
• 설정은 비밀이 아닌 값은 config.json, 민감한 값은 .env로 나눠서 관리하고, Heroku에서는 웹 다이노만 실행해야 해요. 워커 다이노를 실행하면 의도적으로 실패하도록 만들어져 있어요.

WAVE 03 104 pts · 9 comments

파이어폭스 빌드 시간을 17% 줄인 방법, 생각보다 단순했어요

• 파이어폭스는 빌드할 때마다 .webidl 파일 수천 개로부터 C++ 바인딩 코드를 생성하는 WebIDL 코드젠 단계를 거쳐요. 이 과정은 입력이 같으면 출력도 항상 같은 결정론적 작업인데, 지금까지 매번 새로 실행하고 있었던 거예요.
• buildcache의 Lua 플러그인 시스템을 활용해서 이 단계를 캐싱했어요. ccache나 sccache는 컴파일러만 감쌀 수 있는데, buildcache는 Lua 스크립트로 어떤 프로그램이든 캐싱 대상으로 만들 수 있어요. 웜 빌드 기준으로 1분 27초에서 1분 12초로 줄었어요.
• 핵심은 webidl.lua 래퍼가 입력 파일과 출력 파일을 정확히 명시해서 buildcache가 해시를 계산하고 캐시 히트 여부를 판단하게 한다는 거예요. 파이어폭스 빌드 안에는 비슷한 결정론적 코드젠 단계가 더 있어서 같은 방식으로 추가 개선 여지가 있다고 해요.

💡 오늘의 인사이트
